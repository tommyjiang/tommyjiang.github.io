数据标注系列文章（一）

关于深度学习算法的总结网上已经汗牛充栋了，但关于数据标注的总结应该凤毛麟角甚至没人关心。毕竟和算法比起来，标注没什么技术含量。由于目前机器学习的主流仍然是监督学习(supervised learning)，监督学习的一大特点是需要标注好的数据作为输入训练模型。目前的标注需要消耗大量的人力，如何改善标注质量、节约标注成本，是值得深入讨论的问题。

2012. Crowdsourcing annotations for visual object detection

目前找到的最早讨论众包数据标注的文章，作者之一是大名鼎鼎的李飞飞，文章链接在这。

文章主要讨论了众包平台对目标检测算法数据的标注流程。目标检测算法对数据的要求是：

  • 准确性：物体周围有且仅有一个紧贴物体边缘的框(bounding box)。
  • 全面性：图片中所有需要标注的物体都应该有对应的框。

为了保证数据质量，一般的标注方法采用多数表决法(majority voting)，即让多个人标注取交集。本文采用将标注任务分解的方法，将标注任务分解为三个子任务：

  • 画框子任务：在一个物体周围画一个框。
  • 框准确性子任务：检查框画得是否正确。
  • 框全面性子任务：检查是否图中所有待标注物体都有一个框。

对于准确性和全面性子任务，由于只需要回答是和否，因此很容易导致作弊，即不看图片全选是/否或随便选。为防止恶意操作，需要对质量进行验证(validation)。文中采用事先在图片中植入已经有 Ground Truth 的图片来进行检验。

根据统计结果，画框任务的通过率只有 *62.2%*，画一个框平均需要 50.8s。而准确性和全面性的任务平均只需要 21.9s 和 15.3s，由此得出本文的工作流比多数表决法省时间(至少需要画两次)。

标注文档清晰整洁，值得学习。

思考

  1. 只讨论了检测标注任务，未涉及到画线、多边形等任务(分类任务较简单)。
  2. 全面性子任务如果发现有漏的物体，如何有效使画框子任务的人员知道应该画在哪，例如做个标记？
  3. 工钱怎么算，按画框数和检查框数？可以按照完成的平均时间比例设置单价。

2017. Polygon-RNN

CVPR 2017 Best paper 提名，论文链接看这里。搜了搜中文的内容，没有特别好的解读，正好借学习的机会总结。

概述

Polygon-RNN 要解决的是如何对图片进行分割标注，即确定图片中的每一个像素是属于哪一个物体。但本文在处理时，将 pixel-labeling problem 转化为 polygon prediction task，即多边形预测任务——一个封闭多边形内的所有像素属于同一个物体。通过文中方法可以有效提升标注速度(Cityscapes 数据集上提升了 4.7 倍)。

方法

Polygon-RNN 的输入为一个 bounding box，输出为该 bounding box 之内物体的封闭的 polygon。封闭指的是 polygon 任意两个相邻顶点之间均有一条边。一个封闭多边形有很多等价的表示，论文中采用顺时针描述多边形，起点可以任意指定。

Polygon-RNN 中包含一个 CNN 和一个 RNN，CNN 用于提取图像特征，RNN 用于生成 polygon 的顶点。

CNN

CNN 的输入为 224*224*3 的图片，输出为图片的特征(28*28*128)。Backbone 采用 VGG16，从 pool_2，pool_3，conv4_3 和 conv5_3 取特征接 3*3 卷积(pool_2 需要再接 max pooling，conv5_3 需要 bilinear upsample)，分别生成 28*28*128 的特征，合并组成 28*28*512 的特征，再经过 3*3 的卷积生成最终的 28*28*128 的特征。

RNN

RNN 的输入包括 CNN 特征，t-1 和 t-2 时刻的顶点以及起点，输出是 t 时刻的顶点以及该点是否为终点。起点单独处理。
